## CamelCase descriptive tag names

Tag names should be self-explanatory without reading their contents. Use descriptive CamelCase that explains the purpose.

```markdown
// bad
<Step1/>
<DoThing/>
<Process/>

// good
<ExtractMethodSignatures/>
<BuildTypeMap/>
<GenerateOutput/>
```
